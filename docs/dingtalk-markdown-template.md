# DingTalk Markdown 通知模板

## 默认模板

后续钉钉运营通知默认使用 `field-block-v1`。

适用场景:

- 店铺异常通知.
- 账号状况新增/变化通知.
- 需要运营人员快速阅读和处理的列表型通知.

不再默认使用:

- ActionCard.
- 批次号标题.
- 长横线单行字段, 例如 `问题类型 | 日期 | 当前处理 | ASIN | SKU`.
- `明细` 这类低信息量小标题.

## 结构

标题使用人类可读日期, 不带批次号:

```markdown
### 2026年6月26日亚马逊账号状况异常新增通知
```

第一段固定叫 `结论`, 先给数量和范围:

```markdown
**结论**
- 新增/变化: 7 条
- 涉及店铺: 3 个, Artiqua 4条, Soebiz 2条, Wowkk 1条
- 问题类型: 违反受限商品政策 5条, 食品和商品安全问题 2条
- 范围: 美国站, 未解决账号状况异常
```

店铺之间用分隔线隔开:

```markdown
---

#### 店铺: Artiqua
共 4 条 违反受限商品政策: 4 条
```

每条异常使用字段块, 字段顺序固定:

```markdown
1. 问题类型: 违反受限商品政策<br>
   日期: 2026年6月14日<br>
   当前处理: 已发送警告<br>
   ASIN: **B0GDFLJY1F**<br>
   SKU: **SK0042-A-00ORB**
```

## 字段规则

- 顺序固定为: `问题类型`, `日期`, `当前处理`, `ASIN`, `SKU`.
- ASIN 和 SKU 的值必须加粗.
- 普通重复信息不进单条明细, 例如销售风险无变化, 评级无影响.
- 店铺汇总行包含总条数和各问题类型数量.
- 每条异常不要压成一行, 避免钉钉移动端换行难读.
- 代码实现可使用 Markdown 行尾双空格或 `<br>` 触发换行; 文档示例使用 `<br>` 避免仓库出现行尾空格.

## 代码入口

- 当前模板名: `DINGTALK_MARKDOWN_TEMPLATE = "field-block-v1"`.
- 字段块渲染函数: `render_dingtalk_issue_field_block(...)`.
- 完整通知渲染函数: `render_markdown(...)`.
